# Local OpenCode Workflow

This setup uses a 12K local model window. Keep requests, tool output, and replies compact.

- Use built-in read/edit/apply_patch, bash, grep/glob/list, task, todo, LSP, webfetch, and websearch as needed.
- Use `local_code_index` for semantic project search before reading many files.
- Use `local_dev_tools` for project status and safe named checks.
- Use `context7` for library docs and `gh_grep` for public examples only when needed.
- Ask before destructive, global, credential, package-install, or external-directory changes.
